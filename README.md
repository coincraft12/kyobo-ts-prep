# kyobo-ts-prep

> **Kyobo 디지털 자산 관리 시스템 강의 코드를 읽기 위한 최소한의 TypeScript.**  
> TypeScript를 "잘 짜는" 것이 목표가 아니다. 강의에서 나오는 코드를 보고 막히지 않는 것이 목표다.

---

## 이 레포는 무엇인가

Kyobo 강의의 실제 소스코드를 분석해서, 반복적으로 등장하는 TypeScript 패턴만 추려낸 속성 학습 자료다.

**TS 전체 문법을 다루지 않는다.** 강의 코드를 읽는 데 필요한 것만 담았다.

### 이 레포로 할 수 있는 것

- 강의 코드를 보고 "이 줄이 뭘 하는 건지" 이해하기
- `interface`, `class`, `async/await`, 제네릭 같은 패턴을 눈에 익히기
- 강의 따라가는 속도 높이기

### 이 레포로 할 수 없는 것

- TypeScript로 처음부터 코드 짜기
- 심화 타입 시스템 (조건부 타입, 매핑 타입 등) 완전히 이해하기
- TypeScript 공식 문서를 대체하기

## 파일 구성

각 챕터 폴더 안에:
- `README.md` — 개념 설명, 왜 쓰는지, 자주 하는 실수, 체크리스트
- `index.ts` — 실행 가능한 예제 코드 (강의 실제 패턴 기반)

---

## 실행

> **반드시 `ts-study/` 루트 폴더에서 실행할 것.** 챕터 하위 폴더에서 실행하면 `package.json`을 못 찾아 에러난다.

```bash
# ts-study/ 폴더에서 실행
cd ts-study

npm install
npm run ch01   # 챕터별 실행
npm run ch02
# ...
npm run ch08
```

## 학습 순서

1. 챕터 폴더의 `README.md` 읽기 — 개념과 이유 이해
2. `index.ts` 읽기 — 실제 코드에서 확인
3. `npm run chXX`로 실행 — 출력 확인

## 이 자료의 범위 밖

TypeScript의 전체 내용을 다루지 않는다. 아래 항목은 강의 코드를 읽는 데 당장 필요하지 않거나, 직접 코드를 짜는 단계가 되어야 의미가 생기는 것들이다.

- 조건부 타입 (`T extends U ? X : Y`)
- 매핑 타입 (`{ [K in keyof T]: ... }`)
- 데코레이터 (`@Injectable`, `@Controller`)
- `namespace`, `module` 선언
- 타입 선언 파일 (`.d.ts`)
- tsconfig 세부 옵션
